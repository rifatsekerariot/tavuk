IoT Cihaz Kurulum Rehberi

1. MQTT Sunucu Bilgileri

Broker Adresiciftlik.rifatseker.com.tr
Port1883
Protokolmqtt://
Kimlik DoğrulamaYok (Anonim erişim açık)

2. Topic Yapısı

Tüm sensör verileri aşağıdaki ana başlık (topic) formatında gönderilmelidir. {zone_id} kısmı sensörün bulunduğu bölge numarasıdır (1'den başlar).

Sensör Verisi Gönderim Topic'i:

ciftlik/zone-{zone_id}/sensors

Fan Durumu Gönderim Topic'i (SCADA):

ciftlik/scada/fan/{fan_id}

Isıtıcı Durumu Gönderim Topic'i (SCADA):

ciftlik/scada/heater/{heater_id}

Petek (Pad) Durumu Gönderim Topic'i (SCADA):

ciftlik/scada/pad/{pad_id}

3. Payload (JSON Formatı)

Sensörler verilerini standart JSON formatında göndermelidir. Sayısal değerler float veya integer olabilir.

{
  "temp": 24.5,     // Sıcaklık (°C)
  "rh": 60.2,       // Bağıl Nem (%)
  "nh3": 4.2,       // Amonyak (ppm)
  "co2": 580.0      // Karbondioksit (ppm)
}

SCADA cihazları (Fan, Isıtıcı, Petek) durum verilerini standart JSON formatında `status` boolean olarak göndermelidir.

{
  "status": true    // true = Açık (ON), false = Kapalı (OFF)
}

4. Örnek Kod (Python)

import paho.mqtt.client as mqtt
import json

client = mqtt.Client()
client.connect("ciftlik.rifatseker.com.tr", 1883, 60)

# Sensör Verisi Gönderme Örneği
payload = {"temp": 25.0, "rh": 60, "nh3": 5, "co2": 600}
client.publish("ciftlik/zone-1/sensors", json.dumps(payload))

# Fan Durumu Gönderme Örneği
fan_payload = {"status": True}
client.publish("ciftlik/scada/fan/1", json.dumps(fan_payload))

client.disconnect()
